## 7. 可直接用于答辩的话术

- 本项目不是一次性写完的，而是经历了“本地 MVP -> 局域网联机 -> 多游戏大厅 -> 在线服务端 -> 运行时统一重构 -> Survivor 在线同步攻坚”的连续演进。
- 从 Git 记录看，开发节奏非常集中，`2026-07-03` 到 `2026-07-07` 是主要爆发期，基本每天都在同时推进功能、修 bug、清仓库、补文档。
- 分支策略上，前期用 `solo-dev / test / lan-only-apk / unify-game-runtime` 做功能迭代和重构隔离，后期再逐步并回 `main`。
- 后期重点已经不是“能不能联机”，而是“联机架构能不能统一、能不能支持多游戏、能不能继续承载 Survivor 这种实时战斗场景”。
