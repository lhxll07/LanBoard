# LanBoard 当前模块说明

这份文档用于描述当前仓库里各模块的实际职责和现状，不再沿用最初的开工分工表。

## 当前目标

当前仓库已经具备三款桌游、局域网联机、在线房间和独立服务端能力。合并网络重构分支前，现阶段重点是：

1. 保持三款游戏的联机流程稳定。
2. 验证局域网房间发现、加入、断开和过期清理。
3. 保持房间页、游戏页、设置页行为一致。
4. 继续收敛 `RoomPage.qml` 和 `AppController` 的复杂度。
5. 在桌面端和 Android 端都能稳定构建。

## 主要模块

### 1. UI 与页面结构

目录：`qml/`

负责内容：

- 页面结构。
- 卡片和按钮等可复用组件。
- 页面切换和入场动画。
- 各游戏页面的交互展示。

当前页面：

- `HomePage.qml`
  - 展示游戏入口和当前配置。
- `RoomPage.qml`
  - 展示局域网房间、在线房间、手动加入、创建房间、玩家列表、准备和开始游戏。
- `GamePage.qml`
  - 五子棋对局页面。
- `DouDiZhuPage.qml`
  - 斗地主对局页面。
- `FlightChessPage.qml`
  - 飞行棋对局页面。
- `SettingsPage.qml`
  - 修改昵称、默认端口、在线服务器地址等配置。

### 2. 应用协调层

目录：`src/app/`

核心文件：

- `appcontroller.h`
- `appcontroller.cpp`

负责内容：

- 串联 QML、房间、网络、游戏逻辑。
- 处理导航跳转。
- 管理设置持久化。
- 管理加入、退出、准备、开始游戏和返回房间流程。
- 将房间状态变化同步给网络层。

当前已经承担的关键逻辑：

- 五子棋、斗地主、飞行棋的本地和联机启动。
- 游戏结束后自动返回房间。
- 返回房间时清空准备状态。
- 保存昵称、默认端口、最近一次加入房间的 IP/端口、在线服务器地址。

### 3. 房间大厅逻辑

目录：`src/lobby/`

核心文件：

- `roommanager.h`
- `roommanager.cpp`

负责内容：

- 玩家列表。
- 房主和成员身份。
- 准备状态。
- 游戏位和旁观位。
- 当前房间游戏类型。
- 开始游戏条件判断。

当前状态：

- 房间容量为 8。
- 五子棋和飞行棋需要 2 个游戏位玩家。
- 斗地主需要 3 个游戏位玩家。
- 房主固定为游戏位，普通玩家可在游戏位和旁观位之间切换。

### 4. 网络同步

目录：`src/network/`

核心文件：

- `networkmanager.h`
- `networkmanager.cpp`
- `linejsonprotocol.h`
- `linejsonprotocol.cpp`
- `networkaddressutils.h`
- `networkaddressutils.cpp`
- `roomdiscoveryservice.h`
- `roomdiscoveryservice.cpp`

负责内容：

- TCP 主机监听和客户端连接。
- 换行分隔 JSON 消息收发。
- 局域网房间发现和房间发布。
- 本机 IPv4 地址选择。
- 房间状态、准备状态、座位状态、游戏开始和游戏同步消息分发。

当前状态：

- `NetworkManager` 保持作为 QML 和 `AppController` 使用的网络门面。
- `LineJsonProtocol` 负责 TCP 上的换行 JSON 编解码。
- `NetworkAddressUtils` 负责本机 IPv4 选择。
- `RoomDiscoveryService` 负责 UDP 房间发现、房间缓存、去重、排序和过期清理。
- 网络层只负责连接、收发、解析、发现和分发消息，不负责房间规则或游戏规则。

### 5. 游戏逻辑

目录：`src/game/`

核心文件：

- `gamecontroller.h`
- `gamecontroller.cpp`
- `doudizhucontroller.h`
- `doudizhucontroller.cpp`
- `flightchesscontroller.h`
- `flightchesscontroller.cpp`

负责内容：

- 五子棋棋盘、回合、落子、胜负和认输。
- 斗地主发牌、出牌、过牌、牌型判断和私有视角状态。
- 飞行棋掷骰、飞机移动、碰撞、终点和胜负。

当前状态：

- 三款游戏均可在本地运行。
- 三款游戏均已接入联机消息同步。
- 游戏规则留在对应 controller 中，不下沉到网络层。

### 6. 在线服务端

目录：`src/server/`

核心文件：

- `serverapp.h`
- `serverapp.cpp`
- `main.cpp`

负责内容：

- 独立 TCP 服务端。
- 在线房间创建、加入、列表返回。
- 在线房间内玩家、座位、准备和游戏同步。

当前状态：

- 支持多个在线房间。
- 支持五子棋、斗地主、飞行棋在线房间。
- 当前服务端仍内联处理换行 JSON 解析，后续可考虑复用 `LineJsonProtocol`。

## 当前已完成闭环

1. 房主创建局域网或在线房间。
2. 客户端通过发现列表、手动 IP 或在线房间列表加入。
3. 双方或多方查看房间状态和玩家列表。
4. 玩家切换游戏位或旁观位。
5. 游戏位玩家准备。
6. 房主开始游戏。
7. 三款游戏进入对应页面并同步核心操作。
8. 对局结束后返回房间并清空准备状态。
