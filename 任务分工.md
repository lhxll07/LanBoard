# LanBoard 当前模块说明

这份文档不再按最初的 4 人开工计划维护，而是描述当前仓库里各模块的实际职责和现状。

## 当前目标

当前仓库已经完成了五子棋 MVP，现阶段重点是：

1. 保持局域网对战流程稳定
2. 保持房间页、游戏页、设置页行为一致
3. 继续收敛页面结构和状态同步逻辑
4. 在桌面端和 Android 端都能稳定构建

## 主要模块

### 1. UI 与页面结构

目录：`qml/`

负责内容：

- 页面结构
- 卡片组件
- 页面切换和入场动画
- 游戏棋盘绘制

当前页面分工：

- `HomePage.qml`
  - 只展示当前游戏和本地配置
- `RoomPage.qml`
  - 未入房时展示加入房间、创建房间、本地双人
  - 入房后展示房间状态、玩家列表、准备和开始游戏
- `GamePage.qml`
  - 五子棋对局页面
  - 对局结束后通过控制层自动回房
- `SettingsPage.qml`
  - 修改昵称
  - 修改默认端口
  - 展示局域网地址

### 2. 应用协调层

目录：`src/app/`

核心文件：

- `appcontroller.h`
- `appcontroller.cpp`

负责内容：

- 串联 QML、房间、网络、游戏逻辑
- 处理导航跳转
- 管理设置持久化
- 管理加入/退出房间流程

当前已经承担的关键逻辑：

- 开始游戏后进入棋盘
- 游戏结束后自动返回房间
- 返回房间时清空双方准备状态
- 保存昵称、默认端口、最近一次加入房间的 IP/端口

### 3. 房间大厅逻辑

目录：`src/lobby/`

核心文件：

- `roommanager.h`
- `roommanager.cpp`

负责内容：

- 玩家列表
- 房主 / 成员身份
- 准备状态
- 开始游戏条件判断

当前状态：

- 已支持本地与联机共享的玩家列表结构
- 已支持按玩家 ID 更新准备状态
- 已支持一键清空准备状态

### 4. 网络同步

目录：`src/network/`

核心文件：

- `networkmanager.h`
- `networkmanager.cpp`

负责内容：

- 主机监听
- 客户端连接
- 房间状态同步
- 落子同步
- 对局开始 / 结束同步

当前状态：

- 已支持主机 / 客户端 TCP 通信
- 已支持房间状态广播
- 已支持五子棋落子同步
- 已去掉旧的 `new_game` 重开协议，改为回房后重新准备

### 5. 游戏逻辑

目录：`src/game/`

核心文件：

- `gamecontroller.h`
- `gamecontroller.cpp`

负责内容：

- 棋盘状态
- 当前回合
- 落子规则
- 胜负判断
- 认输处理

当前状态：

- 已可本地双人运行
- 已可联机同步
- 已与房间返回流程打通

## 当前已完成闭环

1. 主机创建房间
2. 客户端输入 IP 和端口加入
3. 双方查看房间状态和玩家列表
4. 双方准备
5. 房主开始游戏
6. 双方进入五子棋页面
7. 双方同步落子
8. 出现胜负结果
9. 自动返回房间并清空准备状态

## 后续更值得继续做的方向

- 房间内实时改名协议
- Android 端联机与安装流程继续固化
- 页面拆分，降低 `RoomPage.qml` 和 `AppController` 的复杂度
- 网络错误提示与重连体验优化
