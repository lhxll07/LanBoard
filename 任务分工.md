# LanBoard 任务分工

本文档是项目组的实际开工分工表。目标不是“平均分活”，而是尽快做出一个能演示的 MVP。

## 一、项目目标

项目名称：

- `LanBoard（桌域）`

项目方向：

- 基于 `Qt Quick + QML + C++` 的局域网桌游对战平台

第一阶段只做一个最小闭环：

1. 首页进入大厅
2. 创建房间
3. 加入房间
4. 玩家准备
5. 房主开始游戏
6. 进入五子棋页面
7. 双方局域网同步落子
8. 判断胜负并显示结果

第一阶段不做：

- 登录注册
- 复杂美术
- 多种桌游
- 联机匹配
- 云端正式服务器

## 二、分工原则

- 先做 `Windows 桌面可跑通`，再补 Android 打包。
- 按模块分工，不按“谁顺手就改哪里”分工。
- 每个人都要能独立提交一个可验证的小结果。
- 接口先定死，细节后补，避免互相等。

## 三、推荐落地分工

### 方案 A：4 人小组

- A：界面与交互
- B：局域网通信
- C：房间大厅逻辑
- D：游戏逻辑与集成

这是当前最推荐的分法，适合你们先冲 MVP。

### 方案 B：5 人小组

- A：界面与交互
- B：局域网通信
- C：房间大厅逻辑
- D：五子棋规则
- E：集成、测试、文档

如果有第 5 个人，不要让他“打杂”，直接负责集成和验收，会很有价值。

### C：房间大厅逻辑

- 开始游戏条件判断

### D：游戏逻辑与集成

- 回合控制

第一阶段交付：

- 五子棋棋盘状态类
- 落子接口
- 胜负判断
- 对外暴露给界面层的控制接口

## 五、第一周就按这个节奏推进

### 第一步：所有人统一环境

统一版本：

- Qt：`6.10.3`
- 桌面 Qt：`win64_mingw`
- Android Qt：`android_arm64_v8a`
- JDK：`21`
- Android SDK Platform：`36`
- Android Build-Tools：`36.0.0`
- Android NDK：`27.2.12479018`

### 第二步：每个人先交一个“单点可运行结果”

- A：页面能切换，哪怕数据先写死
- B：两台机器之间能通一条消息
- C：房间状态结构能本地跑通
- D：五子棋本地双人轮流落子能运行

### 第三步：开始联调

联调顺序固定为：

1. `界面 <-> 大厅逻辑`
2. `大厅逻辑 <-> 网络通信`
3. `开始游戏 <-> 五子棋逻辑`
4. `完整双机联机测试`

不要一开始四个人同时改同一条链路。

## 六、先约定这几个接口

为了避免后面反复改，第一阶段先统一下面这些最小接口。

### 房间状态

至少包含：

- `roomId`
- `hostName`
- `playerList`
- `isStarted`

### 玩家状态

至少包含：

- `playerId`
- `playerName`
- `isHost`
- `isReady`

### 游戏状态

至少包含：

- `board`
- `currentTurn`
- `winner`
- `gameOver`

### 网络消息类型

第一版先定这几个：

- `create_room`
- `join_room`
- `player_ready`
- `start_game`
- `place_piece`
- `game_over`

## 七、协作规则

### 1. 不要直接在 `main` 上开发

每个人都从 `main` 拉新分支。

### 2. 开工前先同步

```powershell
git checkout main
git pull origin main
git checkout -b feat/你的分支名
```

### 3. 尽量少跨模块改代码

- 做 QML 的，尽量别顺手改网络层
- 做网络的，尽量别顺手改页面结构
- 如果必须跨模块，先在群里说一声

### 4. 一次提交只做一件事

好的提交示例：

- “完成房间页静态布局”
- “实现局域网 TCP 基础连接”
- “补充五子棋胜负判断”

### 5. 合并前必须满足

- 本地能编译
- 自己改的功能能演示
- 没把别人模块搞挂

## 八、当前建议的人员安排

如果现在就要开工，直接按下面这版最稳：

- 1 号：QML 页面
- 2 号：网络通信
- 3 号：大厅逻辑
- 4 号：五子棋逻辑 + 集成

如果你本人是组长，建议你拿：

- `五子棋逻辑 + 集成`

原因很简单：

- 集成位最了解全局
- 最容易发现接口问题
- 最适合把项目往前推

## 九、验收标准

第一阶段完成，至少要能现场演示：

1. A 机器创建房间
2. B 机器加入房间
3. 双方显示玩家信息
4. 双方准备
5. 房主点击开始
6. 双方进入棋盘
7. 一方落子，另一方同步显示
8. 出现胜负结果

只要这个闭环跑通，你们的项目就已经立住了。
